# Vibe Coding 产品开发范式指南

## 0. 适用范围

本指南用于从 0 到 1 开发一个可运行 MVP 产品，适用于：

* 独立开发者
* 程序员主导的产品开发
* 使用 Codex / AI Agent 辅助编码
* 前端、后端、数据库、AI 能力逐步迭代的项目

核心原则：

* 先明确产品边界，再写代码
* 先做静态界面，再接真实逻辑
* 先完成垂直切片，再扩展完整系统
* 所有 AI 开发任务必须有上下文、范围、验收标准
* 文档和代码同步演进

## 2. 创建 AGENTS.md

`AGENTS.md` 是给 Codex / AI Agent 的长期项目指令。

模板：

```md
# AGENTS.md

## Project Goal

Build a maintainable MVP with clear product scope, clean architecture, and fast iteration.

## Development Principles

- Read relevant docs before coding.
- Do not implement features outside the requested scope.
- Prefer simple, maintainable solutions.
- Avoid unnecessary dependencies.
- Keep UI, business logic, data access, and AI logic separated.
- Use TypeScript where applicable.
- Keep components small and composable.
- Update documentation when behavior changes.

## Product Rules

- Follow docs/PRD.md and docs/MVP_SCOPE.md.
- Do not expand MVP scope unless explicitly requested.
- Respect docs/DECISIONS.md.
- Every feature must have clear acceptance criteria.

## UI Rules

- Follow docs/UI_STYLE_GUIDE.md.
- Follow docs/DESIGN_SYSTEM.md.
- Mobile-first.
- Every major page must handle:
  - loading state
  - empty state
  - error state
  - success state

## Coding Rules

- Inspect existing files before modifying.
- Reuse existing components and utilities.
- Do not rewrite unrelated code.
- Do not introduce new architecture unless necessary.
- After changes, run lint, type check, and tests if available.

## Review Rules

After each task, provide:

1. Summary of changes
2. Files changed
3. How to test
4. Known limitations
5. Whether docs need updates
```

# 五、静态原型阶段

## 1. 实现静态高保真界面

要求：

* 不接真实后端
* 使用 mock 数据
* 页面可以真实跳转
* 核心流程可以点击体验
* 覆盖移动端
* 覆盖 loading / empty / error / success 状态

Codex 任务模板：

```md
# Task: Build Static Prototype

## Context

Read:

- docs/PRD.md
- docs/MVP_SCOPE.md
- docs/UX_FLOW.md
- docs/INFORMATION_ARCHITECTURE.md
- docs/UI_STYLE_GUIDE.md
- docs/DESIGN_SYSTEM.md

## Scope

Build a static high-fidelity frontend prototype.

Do not:

- connect real backend
- add database
- implement authentication
- introduce unnecessary dependencies

## Requirements

1. Implement core pages defined in INFORMATION_ARCHITECTURE.md.
2. Use mock data.
3. Support page navigation.
4. Support mobile-first layout.
5. Include loading, empty, error, and success states.
6. Use reusable components.
7. Keep business logic minimal.

## Acceptance Criteria

- App runs locally.
- Core pages are viewable.
- Navigation works.
- No TypeScript errors.
- No obvious mobile layout overflow.
- Components are reusable.
- Mock data is isolated from UI components.
```

# 七、垂直切片开发阶段

## 1. 切片原则

不要按技术层拆任务：

```txt
错误方式：
先做所有数据库
再做所有接口
再做所有页面
````

按用户价值拆任务：

```txt
正确方式：
用户可以创建一个对象
用户可以查看对象列表
用户可以进入详情
用户可以编辑核心信息
用户可以完成一次核心业务动作
```

每个切片必须包含：

* UI
* 状态
* 数据
* 交互
* 测试
* 验收
